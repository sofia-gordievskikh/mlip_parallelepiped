# Заметки: коробка и параллелепипед в inner box search

Короткое объяснение для студента. Формулы простые, специально без строгих
доказательств - это конспект к коду, а не статья.

## Задача

Есть MILP-подобная задача:

```
max  c·x
s.t. A x <= b
     l_bounds <= x <= u_bounds
     x_i целое для i из int_idx
```

Прямо решать MILP дорого. Идея inner box search: вписать в допустимую область
LP-релаксации широкую **коробку**, сдвинуть её центр вдоль `c`, и взять
целочисленный угол коробки как быстрый кандидат в решение.

## Кубовая (axis-aligned) область

Коробка задаётся нижним и верхним углом: `x in [l, u]`, стороны параллельны осям
координат. Переменные LP - `l` и `u` (по `n` штук).

**Опорное ограничение.** Максимум линейной функции `a_k·x` на коробке достигается
в вершине:

```
max_{x in box} a_k·x = sum_i ( a_ki >= 0 ? a_ki·u_i : a_ki·l_i )
```

Требуем этот максимум `<= b_k` - тогда вся коробка внутри допустимой области.

**Целочисленная ширина.** Чтобы внутри коробки точно был целый уровень по оси
`i`, требуем `u_i - l_i >= tau` для `i из int_idx`.

**Цель.** Максимизируем `c·(l + u) = 2·c·center` - тянем центр коробки вдоль `c`.

Всё это - обычная LP с `2n` переменными (см. `solve_box_lp`).

## Параллелепипед (повёрнутый базис)

Меняем оси коробки на произвольный **фиксированный** базис `B` (столбцы -
направления рёбер):

```
P = { center + B·t : t in [-ext, ext] }
```

Переменные - `center` (n) и полудлины `ext` (n) >= 0. При `B = I` это ровно
коробка.

Так как `B` фиксирован, `a_k·B_i` - константы, и «все вершины допустимы»
остаётся **линейным**:

```
a_k·center + sum_i |a_k·B_i| · ext_i <= b_k
```

Аналогично параллелепипед держим внутри глобальных границ по каждой координате
`j`:

```
center_j + sum_i |B_ji| · ext_i <= u_bounds_j
center_j - sum_i |B_ji| · ext_i >= l_bounds_j
```

Целочисленная ширина в исходном пространстве по оси `i`:

```
2 · sum_j |B_ij| · ext_j >= tau
```

**Цель:** `c·center + gamma·sum(ext)` (`gamma = 0` - чистый center; `gamma > 0`
слегка поощряет объём).

## Что меняется по сравнению с кубом

- Переменные: `(l, u)` заменяются на `(center, ext)`, плюс задаётся матрица `B`.
- В ограничениях `linprog` появляются множители `|a_k·B_i|` и `|B_ij|` вместо
  прямого разбиения по знаку `a_ki`.
- В наклонной допустимой области требуемую ширину `tau` дешевле набрать **вдоль**
  области, а не по координатным осям. Поэтому куб может стать несовместным
  (`tau` слишком велик для узкой полосы), а параллелепипед, повёрнутый под наклон,
  всё ещё влезает и даёт кандидата с бОльшим `c·center`.

Иллюстрация - `docs/figures/cube_vs_parallelepiped.png` (график objective(tau) и
момент, где куб становится infeasible).

## Ограничения подхода

- `B` задаётся руками (или подбором угла), автоматического выбора базиса тут нет.
- Кандидат - эвристика: округление углового целочисленного уровня, оптимальность
  MILP не гарантируется.
- Всё живёт в LP-релаксации, интегральность проверяется только у финального
  кандидата.
